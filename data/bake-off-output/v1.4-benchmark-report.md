### Known v1.4 issues surfaced (not blocking, queue for v1.5)
1. **`_load_authority_subset()` filter is too narrow** — Charan Singh, Sharad Pawar, V.K. Krishna Menon, Ram Jethmalani are in v1.4 authority but get filtered out of per-PDF authority subsets. Agents can't resolve names they don't see. **Editorial recovery**: validator's `recommended_thinker_already_in_authority` correction catches all of these at validation time (visible in val_corr counts), so the final records are correct — the names just appear in `recommended_authority_additions[]` with a "should resolve to X" correction stamp rather than as resolved mentions inline.
2. **Model under-resolution recurs** — even when authority subset DOES include a name, the model sometimes emits it as a rec_auth instead of resolving. Same pattern Opus flagged on v1.3 (Mangalore meta.b emitted 7 rec_auth that should have resolved; meta.a got 5 right with the same prompt + subset). Self-consistency tiebreak would resolve.
3. **Krishnamachari namespace collision** — V.T. Krishnamachari (Planning Commission VP), T.T. Krishnamachari (Finance Minister), B.T. Krishnamachari (1926 agriculture report author per v1.4 curation). Authority needs disambiguating notes; agents correctly refused to auto-collapse but downstream graph synthesis will need to handle these as distinct nodes.
